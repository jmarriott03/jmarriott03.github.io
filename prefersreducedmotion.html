prefers-reduced-motion and hardware performance

  1. Uses animations normally.
  2. Disables or simplifies them if prefers-reduced-motion is set (which can be used as a proxy for “less powerful” or accessibility-conscious environments).

@media (prefers-reduced-motion: reduce) {
*, *::before, *::after {
animation: none !important;
transition: none !important;
scroll-behavior: auto !important;
}
}

Notes:

  • CSS prefers-reduced-motion is supported in all modern browsers.
  • It does not directly assess hardware performance, but many systems (especially mobile or battery-saving modes) automatically set it when performance is low.
  • For deeper detection of device performance (like FPS or GPU info), you’d need JavaScript, but even then, it’s an estimation and not very reliable across devices.

Examples of “low-power devices” or scenarios

  1. Mobile phones (especially older or mid-tier models)
    • Devices running Android Go or older iPhones.
    • Budget Android devices with limited CPU/GPU power.
    • When in battery saver mode, some devices auto-enable reduced motion.
  2. Tablets
    • Entry-level iPads or Android tablets, especially in power-saving mode.
    • Kids’ tablets or e-readers with limited graphics capability.
  3. Laptops in battery saver mode
    • Windows, macOS, or Linux laptops often reduce motion automatically when:
      • Battery drops below a certain threshold.
      • Power-saving features like “Battery Saver” or “Low Power Mode” are on.
  4. Wearables or embedded devices
    • Smartwatches (like older Apple Watch models).
    • Smart displays or IoT interfaces with stripped-down web UIs.
  5. Accessibility configurations
    • Users on any device (desktop or mobile) may enable “Reduce Motion” via system settings:
      • macOS/iOS: System Preferences > Accessibility > Display > Reduce motion
      • Windows: Settings > Ease of Access > Display > Turn off animations
      • Android: Settings > Accessibility > Remove animations

Why This Matters

Because of the variety of conditions and user choices, using prefers-reduced-motion isn’t just about “performance.” It’s about respecting:

  • Device limitations
  • User preferences
  • Power-saving contexts
  • Accessibility needs

It's the safest proxy available in CSS for deciding whether to reduce or disable animations.
