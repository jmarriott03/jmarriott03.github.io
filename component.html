Create a new 'Text and animation' component in the CMS to support HTML5 embeds

1. Screen readers

  • Use ARIA roles and labels:
    • Provide aria-label or aria-labelledby to describe the content.
    • If the animation conveys meaningful information, make sure it's described in text or through aria-describedby.
  • Hide decorative animations:
    • If the animation is purely decorative, use aria-hidden="true" or role="presentation".
  • Descriptive fallback content:
    • For iframes or SVGs, include alternative text or an accessible summary outside the iframe/SVG.

2. Reading mode (reader view in browsers)

  • Ensure semantic HTML outside of the animation:
    • Reader mode typically strips non-semantic content. Ensure key messages are conveyed through semantic HTML.
  • Avoid putting key content inside inaccessible canvases or iframes without a fallback.

3. Keyboard access

  • Make sure iframe content is keyboard navigable:
    • Test that interactive content can be accessed and controlled using a keyboard.
  • Focus management:
    • Use JavaScript to manage focus appropriately when users interact with embeds.

4. ARIA roles and states

  • Role and live regions:
    • Use role="img" with aria-label for informative SVGs.
    • Use aria-live="polite" or assertive for live updates like loaders.
  • Avoid overuse of ARIA:
    • Only use ARIA where necessary and meaningful.

5. Graceful degradation/fallbacks

  • No-JS fallback:
    • Provide content alternatives for users without JavaScript.
  • Reduced motion preference:
    • Respect the prefers-reduced-motion media query. Avoid auto-playing or looping animations if the user has requested reduced motion:
      @media (prefers-reduced-motion: reduce) {
      .animated-element {
      animation: none !important;
      transition: none !important;
      }
      }
  • Timeouts and pause controls:
    • Offer controls to pause or stop long or looping animations.

6. Mobile responsiveness

  • Ensure responsive embed scaling using max-width: 100% and height: auto.
  • Touch targets should be at least 44 by 44 pixels for usability.

7. Content in iframes

  • Title attribute:
    • Always use a descriptive title:
      <iframe src="animation.html" title="Illustration of recycling process" width="600" height="400" allowfullscreen loading="lazy"></iframe>

      This iframe uses loading="lazy" and allowfullscreen to enhance performance and control.

  • Accessible iframe content:
    • Make sure the HTML inside the iframe is also accessible.

8. SVG-specific considerations

  • Text inside SVGs:
    • Use <title> and <desc> for names and descriptions.
  • Role and focusability:
    • focusable="false" for decorative SVGs, tabindex="0" for interactive ones.

9. Lottie animations or JSON-based animation libraries

  • Not natively accessible — use caution.
  • Provide accessible alternatives like text or images.
  • Don’t rely on them for critical messaging.

10. Design consistency

  • Ensure all components use consistent styling, spacing, fonts, and color themes per CMS standards.

11. User feedback

  • Consider adding a feedback form or contact point for reporting accessibility issues.

Implementation Checklist

Before implementing a new text and animation component in the CMS, ensure you've addressed all these requirements:

6Screen reader accessibility with proper ARIA attributes

6Content is accessible in reading mode

6Full keyboard navigation support

6Appropriate ARIA roles and states

6Fallback content for users without JavaScript

6Respect for reduced motion preferences

6Mobile responsive design

6Accessible iframe implementation

6SVG accessibility optimizations

6Alternatives for Lottie or JSON animations
